# PDF_to_HTML

Интерактивная HTML-версия добавленного документа `input1.pdf`. Все страницы исходного PDF преобразуются в компактный и удобный для просмотра HTML без визуальной потери исходной векторной верстки. HTML показывается в браузерном просмотрщике с постраничной навигацией.

## Краткие OKR проекта


**Objective:** перенести добавленный PDF в компактный и удобный для просмотра HTML без визуальной потери исходной векторной верстки.

1) input1.pdf - исходный файл для обработки
2) handler.py - оркестратор, выполняет преобразование исходного файла в файлы для HTML. Выполняется запуск скриптов:
- извлечение из pdf изображений в RAM для последущей обработки скриптами обработки;
- распознавание содержимого текста и форматирования на изображениях в RAM память;
- преобразование и упаковка в Brotli quality 11 содержимого текста и форматирования из RAM  и сохранение в `data/`;
- распознавания векторных схем на изображениях в RAM память `data/`;
и преобразование векторных схем из RAM в формат SVG и сохранение в `data/`;
- распознавания иллюстраций на изображениях в RAM память `data/`;
- преобразование иллюстраций из RAM в формат avif с качеством 65% и сохранение в `data/`;
- создание адаптивного HTML-просмотрщика с кнопками, вводом номера страницы и навигацией стрелками клавиатуры.
3) Файловая структура:
- `src/` подчиненные скрипты оркестратора;
- `data/locales/` в папке содержатся необходимые многоязычные данные для HTML о текстовом наполнении. 
- `data/pages/` в папке содержатся необходимые данные для HTML о структур страниц и форматировании.
- `data/diagram/` в папке содержатся векторные схемы для HTML.
- `data/images/` в папке содержатся иллюстрации для HTML.
4) server.py - сервер для запуска HTML файла

## Запуск

Браузер автоматически распаковывает ответ. Для перехода между страницами можно использовать кнопки, номер страницы или клавиши `←` и `→`.

```bash
python -m pip install -r requirements.txt
python generate.py
python server.py
curl -I http://127.0.0.1:8000/data/pages.json.br
```
