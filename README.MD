# PDF → Single Page Accessible HTML Converter

Проект преобразует PDF-документы в **один самодостаточный HTML-файл**, который можно многократно открывать в браузере и использовать с расширениями перевода, озвучивания текста, поиска и копирования.

Основная цель — один раз обработать PDF, включая сканированные, распознанные и смешанные документы, а затем работать **с одной HTML-страницей**, а не с PDF и не с набором отдельных файлов. 

AVIF images + compact delta JSON + Brotli:
визуальный слой страницы → AVIF
текстовый слой слов      → delta JSON
сжатие JSON              → Brotli

    data/

    locales/
        ru.json.br          // массив массивов строк (русский)
        en.json.br          // массив массивов строк (английский)

2) полное изображение страницы:
data/images/page1.avif
data/images/page2.avif
data/images/page3.avif

portrait  → максимум 1080×1920
landscape → максимум 1920×1080

3) json
- data/pages.delta.json.br:
    размеры страниц
    пути к AVIF
    словарь уникальных слов
    координаты word boxes
    OCR confidence
    pages.delta.json → pages.delta.json.br
    После распаковки структура такая:
    {
    "v": 1,
    "format": "word-delta",
    "dict": {
        "words": [
        "Depressurisation",
        "compare",
        "with",
        "water",
        "flow"
        ]
    },
    "pages": [
        {
        "w": 1080,
        "h": 1527,
        "img": "images/page1.avif",
        "words": [
            [0, 310, 522, 124, 12, 92],
            [1, 207, 128, 63, 9, 95],
            [2, 73, 0, 31, 12, 96]
        ]
        }
    ]
    }

- pages.delta.json:
    {
    "dict": {},
    "pages": []
    }
- dict. Словари:
    "dict": {
    "words": [...]
    }

- pages. Массив страниц:
    "pages": [...]

- dict.words. Словарь уникальных слов документа:
    "words": [
    "Depressurisation",
    "compare",
    "with",
    "water",
    "flow"
    ]

- Структура страницы:
    {
    "w": 1080,
    "h": 1527,
    "img": "images/page1.avif",
    "words": [...]
    }

    w. Ширина AVIF-страницы в пикселях:
    "w": 1080
    h. Высота AVIF-страницы в пикселях:
    "h": 1527
    img. Путь к изображению страницы:
    "img": "images/page1.avif"
    Путь считается относительно каталога data/.

- Итоговый URL:
    data/images/page1.avif
    words. Массив слов страницы:
        "words": [
        [0, 310, 522, 124, 12, 92],
        [1, 207, 128, 63, 9, 95],
        [2, 73, 0, 31, 12, 96]
        ]
        
        Структура одного слова:
        В delta-формате слово хранится как массив:
        [wordIdx, dx, dy, w, h, conf]

        Пример:
        [1, 207, 128, 63, 9, 95]

        Расшифровка:
        wordIdx = 1
        dx      = 207
        dy      = 128
        w       = 63
        h       = 9
        conf    = 95

        wordIdx указывает на словарь:
        dict.words[1] = "compare"

- Восстановление координат
    Вместо абсолютных x/y хранятся дельты относительно предыдущего слова.
    Для первого слова:
    previousX = 0
    previousY = 0
    Первое слово:
    [0, 310, 522, 124, 12, 92]
    Восстановление:
    x = 0 + 310 = 310
    y = 0 + 522 = 522
    Второе слово:
    [1, 207, 128, 63, 9, 95]
    Восстановление:
    x = 310 + 207 = 517
    y = 522 + 128 = 650
    Третье слово:
    [2, 73, 0, 31, 12, 96]
    Восстановление:
    x = 517 + 73 = 590
    y = 650 + 0 = 650

    Итог:
    Depressurisation → x=310, y=522, w=124, h=12
    compare          → x=517, y=650, w=63,  h=9
    with             → x=590, y=650, w=31,  h=12

- JavaScript (renderer.js). Логика:
    - Загружаем pages.json.br и нужный locales/ru.json.br (распаковываем Brotli через DecompressionStream или полифил). 
    При отображении страницы N:
    - берём pages[N-1],
    - берём translations[N-1],
    - для каждого элемента text берём строку по textIdx,
    - создаём DOM-элементы с абсолютным позиционированием.

# Обработка содержимого текста
Сохрани параметры символов/слов при обработке текста для каждого слова. Что бы можно было понять на основе какого правила выполнена одна из операций ниже.

1) При распозновании текста использовать `psm=11`. После разпознавания текста, необходимо выполнить разделение содержимого страниц. Добавь кнопку которая будет скрывать подсвеченные области и линии.

2) Найди все линейные объекты на изображении с помоью OpenCV. Выдели линейные объекты красной пунктирной линией.

3) Выдели 10 порядков областей на изображении прямоугольником с номером порядка по середине. Для каждого порядка существует только одна область. посвети области на изображении. Области выделяют пустое пространство на изображении между символами. Пустое пространство не включает символы и линейныхе объекты. Символы должны ограничивать область либо сверху и снизу, либо слева и справа. Номер порядка области отражает ее размер - мощность (протяженность по меньшей стороне, расстояние между символами), область первого порядка самая большая (первая в топе), второго порядка вторая в топе и т.д.
- Область первого порядка. Определи прямоугольную область с самой большой мощностью (протяженность по меньшей стороне, расстояние между символами), внутри которого нет символов. Границами  прямоугольной области служат края страниы (по пикселям);
- Область второго порядка. Определи прямоугольную область с меньшей мощностью чем у области первого порядка, но самой большой из оставшихся. Границами  прямоугольной области служат края страницы и область первого порядка(по пикселям);
- Если мощность области некоторого порядка становится равна 1.1 * Int.av. (Int.av. - средний межстрочный интервал), построение областей завершается.
- Поиск областей и присвоение порядка выполняются раздельно: сначала пустые области рекурсивно используются как структурные границы, затем все найденные области сортируются по мощности. Узкая область может открыть внутри части страницы более мощную область; при этом открытая область получает меньший номер порядка в итоговом рейтинге.

4) Закрась области:
- Каждую область, которая расположена протяженной стороной горизонтально по всей ширине страницы, от левой части изображения до правой;
- Каждую область, которая расположена протяженной стороной вертикально и делит страницу на 2 равные части, то есть захватывают середину изображения (координаты y, расположенные в 47.5-52.5% ширины страницы);
- Пару областей, которые расположены протяженными сторонами вертикально и параллельно друг другу. Они делят страницу на 3 равные части, то есть захватывают часть изображения (координаты y, расположенные в 30-35% ширины страницы и 60-70 %);
- Каждую область, которые расположена протяженной стороной горизонтально между краем изображения и любой из закрашенных областоей с протяженной стороной расположенной вертикально.

5) Выдели текстовые области - текстовые блоки, где:
- Символы ограничены закрашенными областями любого порядка порядка или границами изображения;
- Подкрась текстовые блоки персиковым цветом;
- пронумеруй текстовые блоки с сверху вниз, сначала в левой части изображения, затем в правой;

6) Выдели opencv маркеры слева от начала строк:
- маркер может выглядить как "-", "+", жирная точка, галочка, квадрат, ромб, круг, некоторый набор пикселей по размеру высоты строки, повторяющийся перед началом строк;
- подсвети их желтым цветом;

7) В каждом текстовом блоке разграничить текст на предложения. границы предложений устанавливаются при:
- Наличие точки и следующего слов ничинающегося с прописной буквы. Точка считается концом предложения только в том случае, если она не является частью сокращения. Точка является частью сокращения, если слово перед ней входит в словарь известных сокращений либо соответствует форме краткого имени или инициала. После сокращения предложение продолжается, даже если следующее слово начинается с заглавной буквы;
- символы переходящие из текущего текстового блока в левой части экрана в слудеющий текстовый блок правой части экрана согласно нумерации текстовых блоков могут быть объединены одним предложением;
- Символы из центрально размещенного текстового блока переходят только в текстовые блоки размещенные по центру.  
- изменение левого отступа;
- начало нумерованного или ненумерованного списка.

9) Предложение считается списком и разделяется на части при:
- в строке, где вконце стоит ":" или ";";
- разделяется со строки которая начинается повторяющегося маркера списка ("1", "2", "a", "b" и т.д. с точками и скобками после числа);
- разделяется со строки которая начинается с opencv маркера;
- Разделяется после строки заполненой менее чем на 50% ширины, если в предложении идет перечисление существительных без глаголов;

11) страница HTML:
- Если слово в строке выбрано, все слова слева от него в этой строке тоже считаются выбранными.
- загрузить data/pages.word_select.delta.json.br или .json
- декодировать словарь слов
- восстановить координаты из delta
- нарисовать AVIF-страницу
- наложить word boxes
- построить visualIndex и lineId
- обрабатывать выделение мышью
- копировать выбранный текст через Ctrl+C
- переключать страницы и zoom

# Взаимодействие

1) левый клик по слову выбирает всё предложение;
2) добавь ускорение произношения текста;
3) при озвучивании подсвечивается произносимое слово;
4) После произнесенного предложения или части списка производится повторение, но с переводом на русский языки и озвучиванием на русском языке;
5) Сделать кнопку последовательное чтение документа. Последовательное место начинается с начала документа, если выбран фрагмент текста, начинается с фрагмента текста, если во время чтения выбирается другой фрагмент текста, чтение начинается с него. При наличии перевода, при последовательном чтении после каждого произношения предложения на оригинальном английском или при наличии отдельных частей в предложении, озвучивать перевод на русский язык.
