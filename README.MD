# PDF_to_HTML

Интерактивный HTML-просмотрщик документов с точным позиционированием элементов и многоязычными переводами.

## Запуск

`index.html` нельзя открывать двойным щелчком через `file://`: браузеры запрещают такой странице загружать данные через `fetch`. Запустите HTTP-сервер проекта, который правильно отдаёт готовые Brotli-файлы:

```bash
python server.py
```

Затем откройте <http://127.0.0.1:8000/index.html>.

Сервер добавляет к `*.json.br` обязательные заголовки:

```text
Content-Type: application/json; charset=utf-8
Content-Encoding: br
Vary: Accept-Encoding
```

Браузер автоматически распаковывает ответ. JavaScript не использует несжатые JSON, gzip или fallback на них.

## Генерация данных

Установите зависимости и запустите генератор:

```bash
python -m pip install -r requirements.txt
python generate.py
```

Генератор создаёт только готовые Brotli-файлы с quality 11:

- `data/pages.json.br` — геометрия и элементы всех страниц;
- `locales/ru.json.br` — русские переводы;
- `locales/en.json.br` — английские переводы;
- `data/images/*.avif` — изображения страниц.

Перед генерацией каталоги `data/` и `locales/` пересоздаются, поэтому старые несжатые JSON и gzip-файлы не сохраняются.

## Настройка production-сервера

Для любого сервера/CDN файлы `*.json.br` должны отдаваться без дополнительного сжатия и с `Content-Encoding: br`. Пример для Nginx:

```nginx
location ~ \.json\.br$ {
    default_type application/json;
    add_header Content-Encoding br always;
    add_header Vary Accept-Encoding always;
}
```

После настройки проверьте заголовки:

```bash
curl -I http://127.0.0.1:8000/data/pages.json.br
```

## Компактный lossless-формат данных

`pages.json.br` не использует повторяющиеся JSON-ключи. После автоматической распаковки браузером он содержит позиционные массивы:

```text
[version, fonts, colors, sources, svgs, styles, pageSizes, pages]
```

Повторяющиеся строки шрифтов, цветов, путей изображений и SVG хранятся один раз в таблицах. Стиль представлен как `[fontIdx, size, flags, colorIdx]`, где первый бит `flags` означает `bold`, а второй — `italic`.

Повторяющиеся размеры страниц также вынесены в таблицу `pageSizes`; страница представлена как `[pageSizeIdx, elements]`. Элементы остаются в исходном порядке, поэтому сохраняется порядок наложения:

```text
text  = [0, textIdx, styleIdx, x, y, w, h]
image = [1, srcIdx, x, y, w, h]
svg   = [2, svgIdx, x, y, w, h]
```

Все исходные поля и их значения сохраняются без округления координат или другой потери точности. Генератор перед записью автоматически восстанавливает исходную объектную схему из компактной и проверяет их полное равенство.

Файл каждого языка содержит массив массивов строк, поэтому перевод выбирается по `textIdx`. Все страницы загружаются один раз. При смене языка загружается только соответствующий Brotli-файл переводов, если его ещё нет в памяти.
