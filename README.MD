# PDF_to_HTML

Преобразование сканированных документов в интерактивный HTML с точным сохранением макета и многоязычной поддержкой.

## Ключевые ценности

- **Точность макета** – абсолютное позиционирование каждого слова, полная информация о шрифтах, размерах, начертаниях.
- **Минимальный трафик** – Brotli-сжатие данных, компактный формат переводов (массив массивов без ключей).
- **Мгновенная навигация** – один загружаемый файл для всех страниц, переключение страниц и языков без дополнительных запросов.
- **Простота поддержки** – переводы хранятся отдельно от структуры, редактируются как обычный JSON.
- **Автономный просмотр** (опционально) – генерация самодостаточного HTML со встроенными данными и изображениями.

## Форматы данных

### pages.json

После распаковки — массив объектов страниц. Каждая страница содержит размеры холста и массив элементов.

**Типы элементов:**

| Тип     | Назначение                           | Дополнительные поля               |
|---------|--------------------------------------|-----------------------------------|
| `text`  | Текстовый фрагмент                   | `textIdx`, `font`, `size`, `bold`, `italic`, `color`, координаты, размеры |
| `image` | Растровое изображение                | `src` (путь к файлу), координаты, размеры |
| `svg`   | Векторная графика (встроенный SVG)   | `svg` (строка), координаты, размеры |

**Пример:**
```json
[
  {
    "width": 1240,
    "height": 1754,
    "elements": [
      {
        "type": "text",
        "textIdx": 0,
        "font": "Times New Roman",
        "size": 12,
        "bold": true,
        "x": 100, "y": 200,
        "w": 45, "h": 14
      },
      {
        "type": "image",
        "src": "images/page1_img1.avif",
        "x": 300, "y": 400,
        "w": 200, "h": 150
      },
      {
        "type": "svg",
        "svg": "<svg>...</svg>",
        "x": 50, "y": 500,
        "w": 600, "h": 300
      }
    ]
  }
]

Важно: textIdx — порядковый номер фразы в массиве переводов для текущей страницы.

Файлы переводов
Формат: массив массивов строк без ключей.

[
  ["фраза 1 стр.1", "фраза 2 стр.1", ...],
  ["фраза 1 стр.2", "фраза 2 стр.2", ...],
  ...
]

Индексация: translations[номер_страницы - 1][textIdx].

Преимущества:

Отсутствие ключей-строк минимизирует размер.

Отлично сжимается Brotli/gzip.

Легко редактировать, экспортировать в инструменты перевода.

Технические решения
Сжатие Brotli
Все JSON-файлы отдаются сжатыми Brotli (quality 11).
Brotli даёт выигрыш ~35–40% по сравнению с gzip на наших данных.

Измеренные размеры для 800 страниц (200 слов на страницу):

Файл	JSON (сырой)	gzip	Brotli
pages.json	25.3 МБ	1.5 МБ	0.89 МБ
ru.json	2.1 МБ	152 КБ	121 КБ
en.json	1.4 МБ	136 КБ	118 КБ
Изображения (AVIF)	509 КБ	–	–
Итого (один язык)	–	2.1 МБ	1.5 МБ
Монолитная архитектура
Все страницы одного документа хранятся в одном JSON-файле.
Загружается один раз при открытии сайта. После парсинга доступ к любой странице – микросекунды.

Рендеринг страниц
JavaScript создаёт DOM-элементы с абсолютным позиционированием согласно координатам из pages.json.
Стили (шрифт, размер, цвет, жирность) применяются inline.

Смена языка
При переключении языка:

Загружается соответствующий *.json.br из locales/ (если ещё не загружен).

Для текущей страницы из translations[page-1] берутся строки по textIdx.

Текст в DOM-элементах обновляется без пересоздания структуры.

Изображения и SVG
Растровые изображения хранятся в data/images/ в формате AVIF (качество 85%).

Векторные схемы встроены в pages.json как строки SVG.

При рендеринге SVG вставляется через innerHTML, изображения — через <img src="...">.

Сравнение производительности
Тест на 800 страницах, 200 слов на странице. Доступ к 100 случайным страницам.

Метод	Размер загружаемых данных	Время загрузки + доступ
Монолит (Brotli)	1.5 МБ	~29 мс загрузка, 0.14 мкс на страницу
Постраничные запросы (800 файлов)	~570 КБ суммарно	~6.5 мс на страницу (в сотни раз медленнее)
В реальной сети постраничная загрузка 800 файлов добавит ~800 RTT задержек, что сделает навигацию неприемлемо медленной.

Автономный режим
generate_standalone.py создаёт index_standalone.html, в который встроены:

все данные pages и translations как JavaScript-переменные,

все изображения в формате base64 (AVIF/JPEG/WebP).

Плюсы:

Не требует сервера, открывается двойным щелчком.

Минусы:

Большой размер файла (для 800 страниц — десятки МБ).

Рекомендуется только для тестирования или небольших документов.

Для обхода проблем с AVIF в BytesIO можно использовать временный файл или переключиться на JPEG/WebP (настраивается в скрипте).

Масштабирование
Для документов более 10 000 страниц монолит может превысить 4–5 МБ сжатых данных.
Рекомендованное решение:

Разбить pages.json на чанки (pages_chunk_1.json.br, pages_chunk_2.json.br...).

Реализовать ленивую загрузку: текущий чанк + предзагрузка следующего.

Переводы и изображения остаются отдельными файлами и также могут быть разбиты.

Архитектура позволяет такое разделение без изменения логики рендеринга.

Зависимости
Для работы Python-скриптов генерации требуются:

Python 3.8+

Pillow >= 9.4.0 (с AVIF-плагином или без)

pillow-avif-plugin (опционально, для AVIF; иначе можно использовать JPEG/WebP)

brotli

Установка:

bash
pip install Pillow pillow-avif-plugin brotli
Для автономного HTML без AVIF можно использовать JPEG — дополнительные плагины не нужны.
