# PDF_to_HTML

Интерактивная HTML-версия добавленного документа `input1.pdf` — **Guidelines for Evaluating Water in Pit Slope Stability**. Все 6 страниц исходного PDF преобразуются в масштабируемые SVG и показываются в браузерном просмотрщике с постраничной навигацией.

## Краткие OKR проекта

**Objective:** перенести добавленный PDF в компактный и удобный для просмотра HTML без визуальной потери исходной векторной верстки.

1. **KR1 — анализ:** определить структуру, число страниц и доступные графические операции `input1.pdf`.
2. **KR2 — преобразование:** детерминированно преобразовать все 6 PDF-страниц в SVG и упаковать данные в Brotli.
3. **KR3 — интерфейс:** предоставить адаптивный HTML-просмотрщик с кнопками, вводом номера страницы и навигацией стрелками клавиатуры.
4. **KR4 — проверка:** проверить повторную генерацию, структуру данных, HTTP-заголовки и открытие всех SVG-страниц.

## Запуск

Сначала создайте производные HTML-данные из исходного `input1.pdf`, затем запустите сервер:

```bash
python -m pip install -r requirements.txt
python generate.py
python server.py
```

`index.html` нельзя открывать двойным щелчком через `file://`: браузеры запрещают такой странице загружать данные через `fetch`. После запуска сервера откройте <http://127.0.0.1:8000/index.html>.

Сервер добавляет к `*.json.br` обязательные заголовки:

```text
Content-Type: application/json; charset=utf-8
Content-Encoding: br
Vary: Accept-Encoding
```

Браузер автоматически распаковывает ответ. Для перехода между страницами можно использовать кнопки, номер страницы или клавиши `←` и `→`.

## Повторная генерация HTML-данных из PDF

Установите зависимость и запустите генератор из корня проекта:

```bash
python -m pip install -r requirements.txt
python generate.py
```

### Правило работы с производными файлами

`input1.pdf` и `generate.py` — источники истины. Каталоги `data/` и `locales/` являются производными артефактами, исключены через `.gitignore` и **не должны редактироваться вручную или добавляться в Git**. Любое изменение формата или содержимого выполняется только в `generate.py` либо в исходном PDF, после чего локальные артефакты пересоздаются командой `python generate.py`.

Генератор:

- читает страницы и FlateDecode-потоки из `input1.pdf`;
- преобразует поддерживаемые PDF-команды контуров, заливки, обводки и матриц в SVG;
- пересоздаёт каталог `data/` и файл `data/pages.json.br` с Brotli quality 11;
- удаляет устаревший каталог `locales/`, потому что исходный PDF не содержит переключаемых языковых слоёв.

Исходный PDF создан с помощью **Microsoft Print To PDF**: буквы в нём сохранены как векторные контуры, а не как текстовый слой. Поэтому HTML точно масштабирует исходную графику, но текст документа нельзя выделить или искать средствами браузера.

## Формат данных

После автоматической распаковки браузером `data/pages.json.br` содержит компактный позиционный массив:

```text
[version, svgs, pageSizes, pages]
```

- `version` — версия формата, сейчас `2`;
- `svgs` — SVG-разметка каждой страницы;
- `pageSizes` — таблица уникальных размеров страниц;
- `pages` — пары `[pageSizeIdx, svgIdx]` в исходном порядке.

## Настройка production-сервера

Для любого сервера/CDN файлы `*.json.br` должны отдаваться без дополнительного сжатия и с `Content-Encoding: br`. Пример для Nginx:

```nginx
location ~ \.json\.br$ {
    default_type application/json;
    add_header Content-Encoding br always;
    add_header Vary Accept-Encoding always;
}
```

После настройки проверьте заголовки:

```bash
curl -I http://127.0.0.1:8000/data/pages.json.br
```
